## 1. プロダクト概要

**プロダクト名（仮）：** Verse Memory  
**一言で：** UEFN開発者がAIツールと作業する際に、検証済みのVerse制約・パターンを自動注入するナレッジベースSaaS  
**提供形態：** API / MCP Server（Cursor、Claude Code等のAIツールから直接呼び出し可能）  
**初期投資規模：** 数万円〜10万円（APIホスティング費用程度）

## 2. 市場背景 — 「toA」という新市場

### toAとは
「toC」（消費者向け）でも「toB」（法人向け）でもない、**AIエージェントそのものが顧客になる市場**を「toA」（to Agent）と呼ぶ。2026年に入り、AIエージェントの普及が加速する中で、エージェントが自律的に動くために必要な周辺サービス（メール、記憶、ブラウザ操作、決済等）が200件以上登場している。

### エージェントの「5つの生存条件」
toAサービスは以下の5条件＋運用拡張レイヤーの二層構造で整理できる：
1. **存在証明**（ID、メール）
2. **実行環境**（サンドボックス、GPU）
3. **操作手段**（ブラウザ操作、外部接続）
4. **記憶**（長期記憶、コンテキスト管理）← Verse Memoryはここ
5. **経済活動**（決済、マーケットプレイス）

### Verse Memoryの位置づけ
「記憶」レイヤーの中でも、**特定ドメイン（UEFN/Verse）に特化した検証済み知識ベース**という独自ポジション。汎用記憶サービス（Mem0等）とは「品質保証付きドメイン知識」で差別化。

参考元：paji.eth氏の記事「Claudeが加速させる『toA』という新潮流」

## 3. 解決する課題

### 一次ペイン：「AIが毎回Verseの制約を忘れる」
UEFN開発者がCursorやClaude CodeでVerseコードを書く際、AIは：
- 非推奨になった構文（`no_rollback`等）を提案する
- 存在しないAPIを平気で生成する
- デバイスの正しい接続方法を知らない
- 以前のセッションで解決した問題を次のセッションで再び間違える

**結果：** 開発者は毎回同じ説明を繰り返すか、AIの出力を逐一検証する必要がある。

### 二次ペイン：「正しいVerse情報が分散・不足している」
- 公式ドキュメントは不完全で分散
- コミュニティの情報は断片的
- AIの学習データには古い情報が混在
- 実機検証しないと正しいかわからない

### Verse Memoryが提供する解決
AIツールがVerse Memoryに接続することで、**AIが最初から「Verseに詳しい状態」で応答する。** 開発者は説明の繰り返しから解放され、AIの出力精度が大幅に向上する。

## 5. 競合・差別化

### 汎用記憶サービス（Mem0等）
- 汎用的な会話記憶の保存・注入
- ドメイン特化の知識はない
- 「検証済み」の品質保証がない

### Verse Memory の差別化ポイント
| 観点 | 汎用サービス | Verse Memory |
|---|---|---|
| 対象 | あらゆる会話 | UEFN/Verse開発に特化 |
| データ品質 | ユーザー依存 | 実機検証済み・バージョン管理付き |
| 検知方法 | キーワードのみ | キーワード＋コードパターンマッチング |
| 更新 | 手動 | UEFNバージョン更新に追随 |

### 公式ドキュメント / コミュニティWiki
- 人間が読むための形式（AIが消費しにくい）
- 網羅性が不十分
- 「AIがどう間違えるか」の情報がない

### Verse Memoryの独自価値
**「AIが間違えるポイント」を知っている。** 単なるドキュメントではなく、AIの典型的な誤りパターン（`common_ai_mistake`）を含む。これにより、AIが間違える前に正しい情報を注入できる。

## 6. プロダクトアーキテクチャ

### 三層構成

```
【データ層】検証済みVerse制約・パターンのナレッジベース（JSON）
     ↓
【API層】キーワード検索 + コードパターンマッチングで関連知識を返す
     ↓
【接続層】MCP Server として Cursor / Claude Code から直接利用
```

### データレコードの構造（1件あたり）
各レコードは以下の情報を持つ：
- **trigger：** いつこの知識を引くか（キーワード＋コードパターン）
- **problem：** 何が問題か＋AIの典型的な間違い
- **solution：** 正しい対処法＋コード例
- **verification：** 検証ステータス・検証者・対応UEFNバージョン
- **severity：** critical（動かない）/ warning（意図しない挙動）/ info（効率Tips）
